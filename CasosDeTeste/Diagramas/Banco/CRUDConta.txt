Use Case - CRUDConta.
Brief Description 
"Allows the inclusion, editing, deleting and searching a student".
System: System.
Primary and Secondary Actors
Primary Actors: Bancario.
Secondary Actors: Cliente.
Main Flow - AddConta.
Bancario starts Use Case.
	Cliente informs the attributes to the Bancario.
	Bancario enters attributes (String nome, String CPF, Date  dataNasc) of Cliente on MainForm.
	Bancario enters attributes (int agencia, int numConta) of Conta on MainForm.
	Bancario  selects "CadastrarConta" on MainForm.
	System validates attributes of Cliente.
	System validates attributes of Conta.

	
	If ["Inconsistência"]
		System  returns "Dados incorretos" to Bancario.
	Else
		System  saves attributes of Cliente.
		System  saves attributes of Conta.
		System  returns "Cadastro efetuado com sucesso" to Bancario.
	EndIf


Bancario finishes Use Case.


Alternative Flows 
Alternative Flow 01 - DeletarConta.

	Cliente informs the attributes (CPF) for the Bancario.
	Bancario  selects "EncerrarConta" on MainForm.
	System validates the attributes (CPF) of Cliente.

	If ["Cpf Inválido"]
		System  returns "CPF Inválido" to Bancario.
	Else
		System  retrieves the informations of Cliente.
	EndIf 

	If ["Cliente Inexistente"] 
		System  returns "Cliente inexistente" to Bancario.
	Else
		System  displays the attributes of cliente on MainForm.
	EndIf
	
	Cliente informs the attributes (Senha) for Bancario.	
	System validates the attributes (Senha) of Conta.


	If ["senha inválida"]
		System  returns "Senha inválida" to Bancario.
	Else
		Bancario  selects "ConfirmAction" on MainForm.
		System  deletes the informations of Cliente.
		System  returns "Cliente deletado com sucesso" to Bancario.
	EndIf

Alternative Flow 02 - Sacar.
	Cliente  selects "EfetuarSaque" on MainForm.
	Cliente  informs the attributes (numConta, String  senha) of Conta on MainForm.
	System  searches  for Cliente.

	If ["Cliente inexistente"]
		System  returns  "Cliente inexistente" to Cliente.
	EndIf

	System validates the attributes (senha) of Cliente.

	If ["Senha incorreta"] 
		System  returns "Senha Incorreta" to Cliente.
	Else
		Cliente informs the attributes (Quantia) on MainForm.
		Cliente  selects "EfetuarSaque" on MainForm.
		System  retrieves the attributes (saldo) of Conta.
	EndIf

	If ["Saldo Indisponível"]
		System  returns "Saldo Indisponível" to Cliente.
	Else
		System  saves the informations (Date  data, Double  saldo, Time horario, String numConta) of Transacao.
		System deducts the attributes (Double  saldo) of Conta.
		System  returns "Saque efetuado com sucesso" to Cliente.
	EndIf

Alternative Flow 03 - Transferencia.
	Cliente  selects "EfetuarTransferência" on MainForm.
	Cliente enters the informations (Conta ContaCredito, Conta ContaDebito, Double Valor) of Transferencia on MainForm.
	Cliente informs the attributes (Senha) of Conta.
	System  retrieves the informations of Conta.
	System validates the attributes (senha) of Conta.
	
	If ["Senha inválida"]
		System  returns "senha inválida" to Cliente.
	EndIf

	If ["Saldo Indisponível"]
		System  returns "Saldo indisponível" to Cliente.
	EndIf

	System  displays the informations of Transferencia on MainForm.
	Cliente  selects "confirmAction" on MainForm.
	System deducts the attributes (valor) of Conta.
	System credits the attributes (valor) of Conta.
	System  saves the informations (Conta ContaCredito, Conta ContaDebito, Time hora, Date  data, String Agencia, Double Valor) of Transferencia.
	System  returns "Transferencia efetuada com sucesso" to Cliente.

	Key Scenario 02 - DeleteStudent.
